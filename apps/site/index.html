Product Preview

EvCode is a source-build-first governed runtime for Codex-native work.

The standard channel keeps Codex in final control while governed specialists contribute planning and frontend visual direction only when the task actually benefits from them.

See the workflow Read the runtime baseline

Proof

Evidence stays ahead of claims.

  • Formal baseline families keep standard and benchmark behavior separate.
  • Portable provider configuration works outside the repo and across machine paths.
  • Live specialist validation remains opt-in so benchmark execution stays deterministic.

Workflow

One governed path from request to cleanup.

  1. Freeze requirements and execution plan.
  2. Route ambiguous planning to Claude when needed.
  3. Route visual frontend direction to Gemini when the request is design-heavy.
  4. Return execution authority to Codex for integration, verification, and release hygiene.

Governance

Codex remains the final engineering owner.

Specialist outputs are advisory inputs, not release authority. The governed runtime, proof artifacts, and cleanup receipts make that boundary explicit.
