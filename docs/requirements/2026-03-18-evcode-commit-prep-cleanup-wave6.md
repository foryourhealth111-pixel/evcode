# EvCode Commit Prep Cleanup Wave6 Requirement

## Goal

整理本次 EvCode 多助手增强与回退治理相关改动，清理测试缓存与临时文件，保留需要提交的代码与文档，并给出一份可提交前的最新验证结论。

## Deliverable

- 一套整理后的提交候选改动集合
- 清理完成的仓库工作区（不包含测试缓存和明显临时文件）
- 新鲜的验证结果与治理收据
- 一份面向提交的范围说明

## Constraints

- 不回滚或覆盖用户/仓库中的无关修改。
- 不删除真实的产品代码、测试代码或正式文档，只清理缓存、临时文件和无价值的过程产物。
- 保留本次多助手增强、混合 baseline、统一配置、回退告警与分发可移植性相关成果。
- 提交前必须重新验证，而不是依赖旧结果。

## Acceptance Criteria

- 仓库中的缓存目录、`.pyc` 与明确的临时测试产物被清理。
- 提交范围中的核心代码、配置、测试、文档被保留并可说明其用途。
- 至少完成一轮针对保留改动集的 fresh verification。
- 输出本轮 cleanup receipt、proof/report，并说明当前已准备好进入提交步骤。

## Non-Goals

- 在本轮新增新的产品功能。
- 修复 RightCodes Claude 外部提供方自身的不稳定性。
- 自动执行 Git 提交或推送。
