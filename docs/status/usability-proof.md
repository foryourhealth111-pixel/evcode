# EvCode Usability Proof

## Goal

Prove that EvCode remains usable as both a host-native standard tool and a benchmark-oriented autonomous tool.

## Required Evidence

- standard channel starts with host-native behavior
- benchmark channel closes a one-shot task without follow-up questions
- artifact locations are deterministic and readable
- doctor output clearly reports missing surfaces
- materialized host/runtime sources remain inspectable by operators

## Pass Criteria

- normal users can inspect channel status through the CLI
- benchmark users can inspect adapter summaries and proof artifacts
- docs point to one authoritative runtime story
- source materialization does not hide patch boundaries
